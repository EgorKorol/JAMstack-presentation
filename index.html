JAMstack for Smartym web services

Created by Egor Korol

What is goal?

Main goal: move Smartym web services to JAMstack. Why?

  1. Leave WP for content editing
  2. Easy to deploy and edit
  3. High performance
  4. Cheap start
  5. No backend

Why the JAMstack?

  1. Better Performance
  2. Cheaper, Easier Scaling
  3. Higher Security
  4. Better Developer Experience
  5. SEO and this is all

Best Practices JAMstack apps

  1. Entire Project on a CDN
  2. Automated Builds
  3. Everything Lives in Git
  4. Atomic Deploys
  5. Modern Build Tools
  6. Instant Cache Invalidation

How to do it?

SSR frameworks + Headless CMS = JAMstack

What SSR framework we should use?

Our choose - Nuxt.js

  • Easy to learn
  • Uses native html/js/css
  • Has many plugins and libraries.

What headless CMS we should use?

Our choose - Strapi

  • Open source
  • Self-hosted
  • Customizable
  • RESTful or GraphQL
  • 100% JavaScript
  • Extensible by design

Ideas:

  1. Deploy a Strapi on a company server
  2. Distribute roles in admin panel
  3. Transfer landings to Nuxt.js (using components)
  4. Mock all text using api from Strapi
  5. Set up WebHooks to run CI
  6. Generate static pages at build time

What's profit?

  1. Managers can change text without engaging developers
  2. We have free admin panel
  3. We can create api for anything
  4. Our websites on CDN
  5. Free and easy
